# 高速化

**Source:** Tips/高速化

## 概要

AYA v5でのゴースト開発における主要なパフォーマンス最適化のガイドラインです。以下の3つの観点から最適化方法を説明します。

## 1. 起動時の最適化

インストール済みゴーストが多い環境では、`aya_shiori3.dic`の以下の設定を変更することで起動を高速化できます。

```
// 内部ゴースト情報検索を無効化
IGLIST_MAX = 0

// 通信解析行数を削減（推奨値: 30）
REQUEST_LINES_LIMIT = 30
```

さらに積極的な最適化として、`On_installedghostname`関数全体をコメントアウトしてゴースト情報取得を無効化する方法もあります。

## 2. ランダムトークのパフォーマンス最適化

大量のランダムトークがある場合、ダブルクォートの代わりシングルクォートを使用することでパフォーマンスが向上します。

通常の書き方:
```
RandomTalk {
    "\0\s[0]こんにちは。\e"
    "\0\s[0]今日もいい天気ですね。\e"
    // ...（数百行）
}
```

最適化された書き方（`EVAL()`と`CHR(0x22)`を組み合わせて埋め込み処理を遅延させる）:
```
RandomTalk {
    EVAL(CHR(0x22) + '\0\s[0]こんにちは。\e' + CHR(0x22))
    EVAL(CHR(0x22) + '\0\s[0]今日もいい天気ですね。\e' + CHR(0x22))
    // ...
}
```

シングルクォートで書くことで辞書読み込み時の埋め込み処理が省略され、大量のエントリがある場合に効果が出ます。

## 注意事項

これらの変更は通常の用途では必要なく、予期しない問題が発生するリスクもあります。パフォーマンス問題が実際に発生した場合にのみ適用することを推奨します。
