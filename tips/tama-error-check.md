# 玉を使った辞書エラーチェック

**Source:** Tips/玉を使った辞書エラーチェック

## 概要

YAYAのデバッグツール「玉（tama）」を使って、ゴースト開発中の辞書エラーを検出する方法を説明します。

## ダウンロード先

- **玉改善プロジェクト**: [GitHub Releases](https://github.com/nikolat/tama)
- **旧配布サイト**: "tama (debugger for aya)"

## 手順1（基本の方法）

1. ゴーストを起動する前に、まず玉を起動する
2. ゴーストを起動する → エラーが色付きのテキストで表示される
3. エラーを確認する
   - オレンジ色：警告（基本動作は可能）
   - 赤色：致命的なエラー（ゴーストが機能しなくなる。要修正）

エラーの表示形式:
```
ファイル名(行番号) : error E????
```

## 手順2（別の方法）

1. 起動中のゴーストをすべて終了する
2. AYAのDLL（辞書ファイルではなくDLL本体）を玉にドラッグ＆ドロップする
3. 手順1と同じ方法でエラーを確認する

## 右クリックメニューの機能

| 機能 | 説明 |
|------|------|
| 先頭/末尾へジャンプ | ナビゲーションショートカット |
| クリップボードへコピー | 結果をエクスポート |
| 受信 | 起動中ゴーストの動作表示のON/OFF切り替え |
| リロード | DLLを再ドロップせずに編集後の再チェック |
| 終了 | ツールを閉じる |
