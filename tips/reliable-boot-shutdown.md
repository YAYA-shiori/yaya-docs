# ゴーストの起動と終了に確実に何かする

**Source:** Tips/ゴーストの起動と終了に確実に何かする

## 概要

ゴーストのロード・アンロード時に確実に処理を実行する方法。SAORIの持続的なロードや変数の初期化などに使う。

## 基本的な方法

任意の辞書ファイルに以下のイベントハンドラを追加する：

```
OnGhostLoad
{
  // 起動時の処理をここに書く
}

OnGhostUnload
{
  // 終了時の処理をここに書く
}
```

> **注意:** これらのハンドラに書いたスクリプトは出力を生成しない。ダイアログ（会話）は `OnBoot` などのイベントで別途実装する必要がある。

## より確実な方法

SSPにはロード済みゴーストをメモリに保持するキャッシュ機能がある。ゴーストの開閉時に確実に処理を実行するには、以下も合わせて実装する：

```
OnCacheSuspend
{
  // キャッシュへの退避時の処理
}

OnCacheRestore
{
  // キャッシュからの復帰時の処理
}
```

> 「通常の開発ではここまで考慮する必要はない」とページで述べられている。
