# 基礎設定

**Source:** マニュアル/文法/1.基礎設定

## 概要

YAYAはDLL名に `.txt` を付加した設定ファイル（デフォルト: `yaya.txt`）を必要とする。このファイルはASCIIエンコーディングを使用し、コンマ区切りのコマンドとパラメータで構成される。

**コメント:** `//` 以降の行および `/*` ～ `*/` 間の内容は無視される。

## 設定例

```
dic, basis.dic
charset, UTF-8
charset.dic, Shift_JIS
log, executelog.txt
```

## 文字コード設定

`charset` でデフォルトエンコーディングを指定する（Shift_JIS / UTF-8 / OSデフォルト）。

| 設定キー | 説明 |
|---------|------|
| `charset` | デフォルト文字コード |
| `charset.dic` | 辞書ファイルの文字コード |
| `charset.setting` | 設定ファイルの文字コード |
| `charset.output` | request関数のI/O文字コード |
| `charset.log` | ログファイルの文字コード |
| `charset.file` | ファイルI/Oの文字コード |
| `charset.save` | グローバル変数保存ファイルの文字コード |
| `charset.extension` | 外部DLL通信の文字コード |

## 辞書ファイルの読み込み

辞書ファイルはプログラム動作を定義するスクリプトソースファイル。

| コマンド | 説明 |
|---------|------|
| `dic, ファイル名` | 辞書を読み込む |
| `dicif, ファイル名` | ファイルが存在する場合のみ読み込む |
| `dicdir, ディレクトリ名` | 指定ディレクトリ内のすべてのファイルを読み込む |

## ログ・デバッグ設定

| 設定キー | 説明 |
|---------|------|
| `log, ファイル名` | 実行ログを有効化 |
| `iolog, [on\|off]` | I/Oログの制御（デフォルト: on） |
| `iolog.filter.keyword` | 指定キーワードに一致するリクエストをログから除外 |
| `fncdepth, [深さ]` | 関数呼び出し深さの上限（デフォルト: 32） |
| `looplimit, [回数]` | ループ繰り返し回数の上限（デフォルト: 10000） |

## その他の設定

| 設定キー | 説明 |
|---------|------|
| `basepath, ディレクトリ` | ファイル操作のベースディレクトリ |
| `include, ファイル名` | 外部設定ファイルのインクルード |
| `save.encode, [on\|off]` | セーブファイルの暗号化 |
| `save.auto, [on\|off]` | 変数の自動保存を有効化 |
| `maxlognum, [数]` | エラーログの最大エントリ数（デフォルト: 256） |

## 緊急設定

バージョン Tc555-1+ では、辞書読み込み失敗時のフォールバック設定をサポート。辞書読み込みが失敗すると、nullを返す前に `yaya_emerg.txt` の読み込みを試みる。
