# 里々からの移行

**Source:** StartUp/里々からの移行

## 概要

里々から YAYA への移行ガイド。プログラミングの専門知識ではなく、構文と構造の基本的な違いを理解することが成功の鍵。

プログラミング的な処理に強いため複雑なゴースト開発が楽になる。非プログラマーも個人の相性次第で問題なく使用できる。

## 里々との重要な違い

### 構文の違い

| 項目 | 里々 | YAYA |
|------|------|------|
| イベント定義 | `*BootTalk` | `OnBoot { ... }` |
| テキスト | 引用符不要 | 必ず引用符で囲む `"text"` |
| 演算子・数値 | 全角可 | 半角のみ |
| 改行・スペース | 自動適用 | SAKURAScriptタグを明示的に記述 |

### 変数の違い

- 里々は変数1種類; YAYA は文字列と数値を区別する
- グローバル変数はプレフィックスなし; ローカル変数にはアンダースコア: `_変数名`
- 文字列代入はクォート必須: `変数 = "テキスト"`; 数値代入は不要: `変数 = 1`

### イベント定義の違い

里々は同名イベントを繰り返し書けるが、**YAYAは同じイベント名を2つ以上書けない**。代わりに1つの波括弧内でランダム選択を使う。

## 必須の準備

テンプレートはほぼ必須。SHIORI/3.0設定スクリプトのため。シンタックスハイライト対応の `.dic` ファイルエディタを推奨。

## コード例（里々→YAYA変換）

### 基本セリフ

```
// 里々
*OnBoot
＼0＼s[0]こんにちは。＼e

// YAYA
OnBoot { "\0\s[0]こんにちは。\e" }
```

### 改行

```
// YAYA（明示的なタグが必要）
"\0\s[0]一行目\n二行目\e"
```

### ランダム選択

```
// YAYA（-- で区切る）
OnBoot { "こんにちは。" -- "おっすー。" -- "やっほー。" }
```

### 条件分岐

```
// YAYA
if (_name == "さくら")
{
  "さくらです"
}
else
{
  "さくらじゃない！"
}
```

### SAORI連携

```
// YAYA
FUNCTIONEX("saori/xxx.dll", Argument0, ...)
```

## よくある問題

| 問題 | 原因・対処 |
|------|----------|
| 辞書が読み込まれない | yaya.txt で明示的に辞書を指定する（里々の自動検出と異なる） |
| ゴーストが動かない | 構文エラーは実行を妨げる。エラーログを確認するか「玉」で検証する |
| セーブデータ | 里々の `satori_savedata.txt` に相当するのは `yaya_variable.cfg` |
