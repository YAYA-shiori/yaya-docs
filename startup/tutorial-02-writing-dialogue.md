# チュートリアル2: トークを書いてみる

**Source:** チュートリアル/2.トークを書いてみる

## 辞書ファイルを開く

開発パレットの「開く」ボタンで GHOST フォルダ内の `.dic` ファイルを開く。メインの辞書は `dic_main.dic`。

## 基本構造

OnBoot イベントでコアの構文を確認：

```
OnBoot { "\0\s[0]こんにちは。\e" }
```

| 要素 | 説明 |
|------|------|
| `OnBoot` | イベント関数名（ゴースト起動時に発生） |
| `{ }` | 関数本体の区切り |
| `" "` | 文字列の区切り（セリフには必須） |
| `\0` | キャラクター識別子（メインキャラクター） |
| `\s[0]` | サーフェス表示コマンド（キャラクタースプライト） |
| `\e` | 終了トークン |

## ランダムセリフの作成

`{ }` 内の複数のクォート文字列はランダムに選ばれる：

```
OnBoot { "こんにちは。" ; "おっすー。" ; "やっほー。" }
```

`--` セパレータで要素を組み合わせる：

```
OnBoot { "\0\s[0]" -- "こんにちは。" "おっすー。" -- "\e" }
```

**選択修飾子:**
- `: nonoverlap` — 直前と同じ候補が連続しないようにする
- `: sequential` — 順番に選択する

## カスタム関数

再利用可能な関数を作成する：

```
OnBoot { "\0\s[0]きょうは" -- 天気一覧 -- "です。\e" }

天気一覧 { "晴れ" "雨" "曇り" "雷" }
```

文字列内での呼び出しは `%(関数名)` を使う。

## フォーマットとコメント

- クォートの外のホワイトスペースは無視される
- `//` で1行コメント
- `/* */` で複数行コメント
- 文字列内で行を継続するには末尾に `/` を付ける

## エラーのトラブルシューティング

- 波括弧が閉じているか確認
- 疑わしいセクションをコメントアウト
- デバッグツール「玉」を使用
- `yaya.txt` でログを有効化
- SSP右クリックメニューからスクリプトログを表示

**重要:** 辞書内で同名の関数は1つしか使えない。
